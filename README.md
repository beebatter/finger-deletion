# 指纹识别级联匹配系统 (Fingerprint Recognition)

本项目是一个结合了**深度学习全局特征**与**传统细节点(Minutiae)拓扑匹配**的指纹 1:1 比对验证系统。系统在开源数据集(SOCOFing)上经过 metric learning 微调，具备极高的鲁棒性，能够准确处理带有切割、压痕、模糊的破坏指纹。

## 系统架构

```text
指纹图像 A, B
       │
  ┌────▼────────────────────┐
  │ 第一阶段: 深度学习比对   │  ResNet 特征提取 → 余弦相似度
  │ 高权重 (86%)             │  主导分数，对宏观形变鲁棒性极强
  └────┬────────────────────┘
       │ 
  ┌────▼────────────────────┐
  │ 第二阶段: 细节点精确匹配  │  端点/分叉点 → 几何拓扑比对
  │ 低权重 (14%)             │  防止跨域身份误判，作为严格的辅助验证项
  └────┬────────────────────┘
       │
  ┌────▼────────────────────┐
  │ 动态分数融合             │  加权融合 → 最终相似度 (0~100%)
  │ 判定阈值                 │  默认 > 88% 判定为同一指纹
  └─────────────────────────┘
```

## 快速开始

### 1. 环境准备
请确保环境中安装了相应的依赖包（主要为 `torch`, `opencv-python`, `scikit-image`, `gradio`）：
```bash
pip install -r requirements.txt
```

### 2. 启动 Web 可视化界面 (推荐)
提供交互式指纹上传与特征可视化标注（红圈端点、蓝圈分叉点）：
```bash
python app.py
```
> 服务器运行后，通过浏览器访问 `http://127.0.0.1:7860`（若在云端可使用VSCode自动端口转发）。

**💥 关闭后台 Web 服务（杀进程技巧）**
如果您修改了配置文件需要重启，或遇到端口被占用(`Cannot find empty port`)导致启动失败，请打开终端运行以下任意一条命令强制释放进程：
```bash
fuser -k 7860/tcp
# 或者（如果 fuser 查不到）：
ps aux | grep "[p]ython app.py" | awk '{print $2}' | xargs -r kill -9
```

### 3. 命令行 1:1 指纹比对
无需开启界面，直接在终端中获取两张指纹的判定结果和相似度分布：
```bash
python recognize.py [第一张指纹的图片路径] [第二张指纹的图片路径]

# 示例:
# python recognize.py data/socofing/SOCOFing/Real/100__M_Left_index_finger.BMP data/socofing/SOCOFing/Altered/Altered-Hard/100__M_Left_index_finger_Zcut.BMP
```

### 4. 数据集下载与重新训练
如果你需要使用我们构筑的 SOCOFing 对比学习管道重新训练提取器权重：
```bash
# 下载 SOCOFing 开源数据库
python download_and_setup.py

# 启动训练
python train.py
```

### 5. 自动化系统级安全评测
系统集成了对真实开源残缺数据库的大规模验证流，通过快速交叉匹配正、负样本，一键输出 **综合准确率**、**TPR (合法识别率)** 与核心的 **FAR (防伪造被骗率)** 报告：
```bash
# 交叉抽取 100 对残缺与伪造指纹自动计算表现：
python eval_socofing.py --samples 100
```

## 关键配置 (`config.py`)

权重与判定阈值目前配置在 `config.py` 中的 `CASCADE` 字典内，您可以随时调整：

- **`match_decision_threshold`** (默认 `0.88`): 综合得分大于此数值(88%)，则判定为“匹配成功：同一指纹”。
- **`deep_weight`** (默认 `0.86`): 深度特征所占算分比重。同源指纹因为具有高维度的一致性，通过神经网络后相似度极高。
- **`minutiae_weight`** (默认 `0.14`): 细节点特征所占算分比重。不同源指纹细节点重合率几乎为0，此分保证彻底扼杀“高仿错认”的可能性。

## 技术细节与算法原理

### 1. 图像预处理管线 (Preprocessing Pipeline)
* **CLAHE 自适应增强**: 在图像进入神经网络与传统提取器前，严格执行限制对比度自适应直方图均衡化（CLAHE），有效消除了因手指干湿、按压力度不同造成的背景噪声和光照差，使脊线对比度显著凸显。
* **方向场与 Gabor 滤波**: 预计算指纹脊线的局部方向场，并引入参数动态 Gabor 滤波器进行纹理增强。此举有效连接了断裂的伪脊线并分离了粘连的污点，从源头上净化了骨架。
* **二值化与骨架化 (Skeletonization)**: 结合自适应局部阈值与形态学细化算法，将多像素宽的指纹黑线精简为单像素宽的纯几何拓扑骨架。

### 2. 深度特征提取与度量学习 (Deep Metric Learning)
* **身份正则解析**: 在针对 SOCOFing 进行对比学习时，系统利用正则表达式动态解析图像变体后缀（如 Zcut、Obl、CR 等破损标记），将同一手指的所有合成破损图片强制归类为同一个指纹身份 (Identity)，由此构建鲁棒的正例锚点。
* **损失函数重构**: 抛弃容易导致梯度收敛停滞的标准 `ContrastiveLoss` (基于绝对欧氏距离的L2范数限制)，全盘切换至 `CosineEmbeddingLoss` 并结合 `AdamW` 优化器。直接优化高维特征在空间内的夹角，完美契合指纹这种“宏观纹理结构”的度量尺度。
* **CNN 特征映射**: 采用微调的 ResNet 结构提取指纹主干，并映射缩降至 256 维的鲁棒表征向量。

### 3. 细节点拓扑计算 (Minutiae extraction & Matching)
* **伪特征多级过滤**: 通过交叉数(Crossing Number)测算出所有骨架细节点（端点/分叉点）后，系统会执行极为严苛的 **五层过滤降噪**：
  1. `边界剔除`：忽略切割边缘造成的强制伪截断点。
  2. `ROI 掩码裁切`：清理非指纹有效区域的噪声脏点。
  3. `最小几何间距`：将距离过近的聚簇毛刺特征融合抛弃。
  4. `动态锐度评估`：依据局部 Laplacian 方差淘汰低质量点。
  5. `提取上限截断`：避免污渍块引发巨量伪特征死锁。
* **坐标空间配准机制**: 匹配引擎并不直接进行点云相减，而是利用局部锚点组合先对两张指纹图像作平移(Translate)与旋转(Rotation)仿射配准，再基于几何容差执行贪心匹配寻找真正同源对。

### 4. 非对称级联分数融合 (Asymmetric Fusion Strategy)
* **原理分析**: 深度特征具有极强的“抗毁损宏观抓取能力”(同源指纹往往能跑出 >95% 分数)，但缺乏局部的极端排异性（异源指纹依然能拿到 30-40% 的外观分）。相反，细节点网络拥有极度排他性（异源 0%），但自身识别上限较低（同源通常只能测得提取到的 60-70% 点位）。
* **策略**: 借助此客观表现，建立偏斜严重的加权公式（`深度 86% : 细节点 14%`），确保合法指纹可以稳稳攀升过 88% 的高阈值屏障，同时让非法入侵指纹因细节点 0 贡献，分数永远被压制在及格线之下。
